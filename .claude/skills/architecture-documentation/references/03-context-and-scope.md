# Guidelines for context and scope

- **Purpose**: Record the system scope and context to explain what is and isn't part of the solution
- **Official documentation**: https://docs.arc42.org/section-3/

## Guidelines for recording the business context

- This section should focus on who is using the system and what other systems our application is communicating with
- Create a C4 context diagram to explain the context.

## Guidelines for recording the technical context

- This section should focus on top-level components in the system and what other systems they communicate with.
- Make sure to include technical information like the protocol used for each communication channel.
