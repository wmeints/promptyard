# Guidelines for concepts

- **Purpose**: Record the crosscutting concepts
- **Official documentation**: https://docs.arc42.org/section-8/

## What should go into crosscutting concepts

- Authentication and authorization
- Transaction management
- Event sourcing patterns
- Event-driven patterns
- Observability

## Guidelines for recording cross-cutting concepts

- Create a separate top-level section for each concept
- Keep the descriptions short, use C4 diagrams where needed
