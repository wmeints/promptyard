# Guidelines for introduction and goals

- **Purpose**: Introduce the application, list the key goals, requirements, and quality goals
- **Official documentation**: https://docs.arc42.org/section-1/

## Listing goals

- List the goals for the project in a bullet list ordered by their business value potential.
- Goals should have business value, if they don't, suggest what the business value could be.

## Listing key requirements

- Requirements must be recorded as a list
- Each requirement should focus on functional value rather than some technological theme
- Keep the description concise

## Listing key quality goals

- List the quality goals in a bullet list with a concise description.
- Limit the list to the top-5 quality goals, ask the developer what he wants to switch out if the list becomes longer.
